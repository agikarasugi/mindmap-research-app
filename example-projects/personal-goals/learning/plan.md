# Learning Plan

## Approach

Deliberate practice over passive consumption. For each topic:
1. Read / watch to build a mental model.
2. Build something small to test understanding.
3. Explain it in writing (blog post or journal entry).

## Q1 Focus: Rust

**Goal:** Be comfortable writing idiomatic Rust for CLI tools and small services.

- [ ] Complete all Rustlings exercises
- [ ] Read *The Rust Programming Language* (chapters 1–15)
- [ ] Build a CLI tool: `mnemo` — a command-line flashcard app
- [ ] Write a blog post: "What Rust taught me about ownership"

**Time budget:** ~5 hours/week, mostly Sunday mornings.

## Q2 Focus: Systems Design

**Goal:** Confidently design distributed systems on a whiteboard.

- [ ] Finish reading *Designing Data-Intensive Applications*
- [ ] Practice 1 system design question per week (use Excalidraw)
- [ ] Write up 3 designs as case studies on the blog

## Q3 Focus: Open Source

**Goal:** Land a meaningful contribution to a well-known project.

- [ ] Identify a project that uses skills I want to deepen
- [ ] Start with documentation or a small bug fix
- [ ] Work up to a feature PR

## Metrics

I know learning is working when:
- I can explain the concept from memory without notes.
- I can answer "why not the obvious alternative?" for the key design decisions.
- I've built something using the concept.
